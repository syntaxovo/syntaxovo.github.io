网络通信

  • 1、两台设备之间通过网络实现数据传输

  • 2、网络通信：将数据通过网络从一台设备传输到另一台设备

  • 3、java.net 包下提供了一系列的类或者接口，完成网络通信

  • 网络：

    • 两台或者多台设备通过一定物理设备连接起来构成了网络

    • 1、用于唯一标识网络中的每台计算机/主机

    • 2、查看ip地址：ipconfig

    • 3、ip地址表示形式：点分十进制 xx.xx.xxx

    • 4、 每一个十进制数的范围： 0 -255

    • 5、ip地址组成 = 网络地址 + 主机地址

    • 6、IPV6 是互联网工程任务组设计的用于替代IPv4的下一代协议、

    • 7、 IPv4 最大问题在于网络通信地址资源优先，严重制约了互联网的应用和发展

    • IP不仅能解决网络地址资源数量的问题，而且也解决了多种接入设备连入互联网的障碍

    • IPv4: 4个字节表示，1个字节的范围是0-255

  • 域名：

    • 好处：为了方便记忆，解决记Ip的问题
    • 概念：将IP地址映射成域名，Http 协议

  • 端口号：

    • 1、概念：用于标识计算机上某种特定的网络程序
    • 2、表示形式：以整数形式，端口范围0-65539 =》 2^16 -1
    • 3、0-1024 已经被占用

  • 常见端口号：

    • tomcat : 8080
    • mysql : 3306
    • oracle:1521
    • sqlserver: 1433

通信协议

  • 数据在网络编程中数据的组织形式就是协议

  • Tcp/ip 协议

  • 中文：传输控制协议/ 因特网互联协议,又叫网络通信协议

  • 这个协议是Internet 最基本协议，Internet 国际互联网络的基础，就是由

  • 网络层的Ip协议和传输层的TCp 协议组成

  • 协议模型

    OSI 功能 协议
    应用层 文件传输 Http、ftp、dns
    表示层 数据格式化、加密 无协议
    会话层 解除或者建立与其他节点连接 无协议
    传输层 提供端对端接口 Tcp、Udp
    网络层 为数据包选择路由 IP、ICMP、ARP
    数据链路层 传输有地址的帧错误检测功能 SLIP、CSLIP、PPP
    物理层 以二进制数据形式在物理媒介上传输数据 ISO2110

Socket

  • Socket 介绍
    • 1、套接字（Socket） 开发网络应用程序被广泛采用，
    • 2、通信两端都要有Socket，是两台机器键通信的端点
    • 3、网络通信即为Socket间的通信
    • 4、Socket 允许程序把网络连接当成一个流，数据在两个Socket 间通过IO传输
    • 5、一般主动发起通信的应用成宿属于客户端，等待通信请求的为服务器
  • Tcp 和 Udp
  • 1、tcp
    •  1、使用TCP 协议前，须先建立TCP连接，形成传输数据通道

    •  2、传输前，采用 ”三次握手“方法是可靠的

    •  3、TCP 协议进行通信的两个应用进程：客户端、服务端

    •  4、在连接中可进行大数据的传输

    •  5、传输完毕，需释放已建立的连接，效率低
  • 2、Udp
    •  1、将数据、源、目的封装成数据包，不需要建立连接

    •  2、每个数据包的大小限制在64k左右

    •  3、因无需连接，故是不可靠的
    • 4、发送数据结束时无需释放资源（因为不是面向连接的）速度快

Udp

  • UDP 网络通信
  • 基本流程：
  • 1、核心的两个类/对象 DatagramSocket 与 DatagramPacket
  • 2、建立发送端、接收端
  • 3、建立数据包
  • 4、调用DatagramSocket的发送、接收方法
  • 5、关闭DatagramSocket
  • 基本介绍：
  • 1、类 DatagramSocket 与 DatagramPacket 实现了基于UDP协议网络程序
  • 2、UDP 数据报通过数据报套接字DatagramSocket 发送和接收，系统
  • 不保证UDP 数据报一定能安全送到目的地，也不确定什么时候可以抵达
  • 3、DatagramPacket 对象封装了UDP 数据报，在数据报中包含了发送端的IP地址
  • 和端口号以及接收端的IP 地址和端口号
  • 4、UDP 协议中每个数据报都给出了完整的地址信息，因此无序建立发送方和接受方的连接
